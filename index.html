🏛️ 1. Piliers Fondamentaux

La reprise de données dans ELAP Finance repose sur quatre piliers essentiels qui garantissent le succès de la migration :

🏛️ Pilier 📝 Description 🎯 Objectif
Continuité Maintenir la traçabilité des engagements non soldés Assurer le suivi budgétaire
Conformité Respecter le cadre GBCP et nomenclatures M4/M57 Garantir la réglementation
Optimisation Épurer au maximum avant migration Réduire les volumes
Intégrité Préserver équilibres et lettrages Maintenir la cohérence

🚫 Stratégie 3 : Reprise de l'Antériorité Complète (Non Recommandée)

🎯 Périmètre : Grands établissements souhaitant l'historique complet

📋 Principe : Montants initiaux + DP de cumul pour le déjà liquidé

⚠️ Complexité : Charge de travail très importante

💰 Coût : Élevé en temps et ressources humaines

📊 Exemple de complexité : Grand établissement = 15 000 DP de cumul reprises !

🚨 ATTENTION : Cette stratégie n'est PAS RECOMMANDÉE. Si le client l'exige, il faut alerter l'équipe projet sur les risques et la complexité. Dans tous les cas, la reprise des informations sera incomplète et il faudra consulter l'ancien outil pour certaines données.

🔢 5. Calculs et Exemples Pratiques

🎯 Situation type : Illustration avec un engagement de 1 000 € partiellement exécuté sur l'exercice N (2025)

🏷️ Type 💰 Montant 📝 Description
🏗️ EJ 1 000 € Engagement initial
✅ SF 500 € Service constaté
💳 DP 200 € Demande payée (porte sur SF)

📊 Calculs de Reprise

📐 Calcul 🔢 Formule 💰 Résultat
Reste à certifier EJ - SF 1000 - 500 = 500 €
Reste à liquider EJ - DP 1000 - 200 = 800 €
🔑 Règle d'or : Le plus important est le reste à certifier ! C'est ce montant qui détermine les volumes de reprise.

📋 Cas d'Usage Pluriannuels

🔹 Cas 1 : Reste à Réaliser "Classique"

🎯 Contexte : EJ validé en 2025, marchandise reçue et payée en 2026

📊 Résultat : 2025 montant engagé = 0 | 2026 montant engagé = CP = 100€

2025 :
• Engagement budget +100€
• OD budgétaire -100€ (restes à réaliser)
2026 :
• Extourne OD +100€
• SF → liquidation +100€ CP

🔹 Cas 2 : Reste à Réaliser en AP/AE

🎯 Contexte : Gestion en autorisations de programme

📊 Résultat : 2025 montant engagé = 100€ | 2026 montant engagé = 0, CP = 100€

2025 :
• Consommation AE +100€
• (pas d'impact sur restes à réaliser)
2026 :
• SF → liquidation +100€ CP

🔹 Cas 3 : Rattachement

🎯 Contexte : EJ + SF en 2025, facture reçue en 2026

📊 Résultat : 2025 montant engagé = CP = 100€ | 2026 montant engagé = CP = 0

2025 :
• Engagement +100€
• SF → mandat charge à payer +100€ CP
2026 :
• Annulation MCAP -100€
• Liquidation +100€ CP

🦸 6. BATMAN - Contrôle Qualité des Données

🎯 Objectif : BATMAN est l'outil de contrôle qualité indispensable pour valider les formats de données avant import dans ELAP Finance.

🔗 Accès : https://elapcdp.streamlit.app/

⚠️ Statut : Outil en cours de développement - nouvelles fonctionnalités ajoutées régulièrement

🔧 Module 📝 Description 🎯 Usage 📊 Statut
👥 Validation Tiers Contrôle qualité données fournisseurs/clients Validation format obligatoire ✅ Opérationnel
⚖️ Validation EJ Contrôle qualité engagements juridiques Validation structure et cohérence 🔄 En cours de développement
🔄 Workflows Interface de sélection fonctionnalités Navigation et pilotage ✅ Opérationnel

🔍 Types de Contrôles BATMAN

🏷️ Type 📝 Description 💡 Exemple
📐 Format Types de données (AN, N, date) 'GENEZADFINAZDFIEZ' > AN(4) ❌
📏 Longueur Respect tailles maximales SIRET '34284296000037A' > 14 caractères ❌
🔗 Cohérence Liens entre fichiers Code liaison ADRESSES ≠ GENERALITES ❌
✅ Complétude Données obligatoires Titulaire, Domiciliation non renseignés ❌
🔢 Numérique Format des valeurs 'SA' au lieu de numérique N(2) ❌

🔍 Contrôles Spécifiques aux Tiers

📋 Fichier 🔍 Type de Contrôle 📝 Description 💡 Exemple d'Erreur
TIERS GENERALITES Format et longueur SIRET principal doit respecter AN(14) SIRETPRINC: '34284296000037A' dépasse 14 caractères
TIERS GENERALITES Format numérique DGP DEPENSES doit être numérique N(2) DGP DEPENSES: 'SA' n'est pas numérique
TIERS ADRESSES Cohérence référentielle Code liaison doit exister dans GENERALITES CODE LIAISON 'F-000198468' absent des GENERALITES
TIERS COORDONNEES BANCAIRES Complétude Titulaire et domiciliation obligatoires TITULAIRE et DOMICILIATION non renseignés
TIERS CONTACTS Données obligatoires Civilité et nom contact requis Civilité et Nom contact non renseignés

📊 Rapport de contrôle : BATMAN génère un rapport détaillé au format : Fichier / Type / Ligne / Erreur détaillée

🎯 Navigation intuitive : Menu principal → Sélection fonctionnalité → Upload fichier → Rapport automatique

🚨 Contraintes Critiques BATMAN

  • Format d'Import Obligatoire : Fichiers Excel uniquement, avec en-têtes spécifiques
  • Contrôles Qualité Automatiques : Validation en temps réel des formats, longueurs et cohérences
  • Règles de Cohérence Critiques : Liens entre fichiers obligatoires (ex: tiers/adresses)
  • Données Obligatoires : Champs requis selon référentiel métier

📊 7. Bilan d'Entrée

📋 Définition : Le Bilan d'Entrée permet de reprendre les soldes comptables de l'ancien système vers ELAP Finance, avec des caractéristiques spécifiques assurant la continuité comptable.

🏷️ Caractéristique 📝 Description ⚠️ Contrainte
📅 Exercice Import sur exercice de démarrage ≠ exercice de reprise
📏 Taille limite < 4 000 lignes (cas d'usage standard) Outil standard uniquement
🔓 Indépendance N'empêche pas le démarrage Import possible en cours d'exercice
🔗 Liaisons Lié aux pièces sources (exercice reprise) Lettrage automatique
📄 Format Excel (.xls) OBLIGATOIRE Avec ligne d'en-tête impérative

📋 Structure du Fichier BE

🔤 Colonne 🏷️ Champ ✅/❌ 📝 Description
A Compte ✅ Obligatoire Nomenclature du client (M4 / M9 (OCB/OHCB) / M57)
B Nature auxiliaire ❌ Facultatif C (Client) / F (Fournisseur)
C Tiers ❌ Facultatif Numéro tiers ELAP
D Référence ✅ Obligatoire Correspondance objets reprise
E Montant Débit 📊 Un des deux 0 si montant au crédit
F Montant Crédit 📊 Un des deux 0 si montant au débit
G Type Pièce Origine ❌ Facultatif Mapping SF/TIT
H N° Pièce Origine ❌ Facultatif Si possible en correspondance avec les numéros de SF/TIT repris dans ELAP
I-M Ventilations ❌ Facultatif Org, Nature, Dest, Op, Ana

📋 Formats de Fichiers Détaillés

⚖️ Format EJ/SF

📋 Structure : Fichier Excel avec sections En-têtes / Lignes / Ventilations / SF

🔑 Colonnes clés : Reference externe, Objet, Date pièce, tiers ELAP, Compte Charge, Montant TTC

🔗 Liaisons : Référence externe pour lier EJ/SF, ventilations budgétaires obligatoires

🤝 Format Accords-Cadres

📋 Structure : Fichier Excel multi-onglets (AccordCadre / Attributaires)

🔑 Données : ReferenceAccordCadre, Etablissement, NumeroAC, Description, NatureMarche

👥 Attributaires : CodeTiers, Rôle (Principal/Sous-traitant), PaiementDirect

🏗️ Format Marchés

📋 Structure : Fichier Excel 5 onglets (Marché / Attributaires / Tranches / Lignes / Ventilations)

⚠️ Limitation : API tranches non disponible = import manuel obligatoire

🔄 Statuts tranches : Tranche Ferme / En attente / Affermie

🎫 Format Titres

📋 Structure : Fichier Excel complexe avec ventilations et services faits optionnels

🔑 Spécificités : Support JSON natif, gestion des prix unitaires, codes TVA

💰 Calculs : Montants HT/TTC automatiques, reste à encaisser

🧹 10. Bonnes Pratiques d'Épuration

🎯 Principe fondamental : L'épuration préalable permet de réduire considérablement les volumes de reprise et de démarrer sur une base saine.

🎯 Action 📝 Description 💡 Avantage
🔒 Solder EJ Finaliser engagements qui ne seront jamais finalisés ↓ Volume reprise
❌ Annuler SF Supprimer services qui ne seront jamais liquidés 🧹 Base propre
💳 Payer DP Liquider toutes les DP prises en charge ↓ Complexité
🔗 Lettrer titres Associer aux encaissements ✅ Cohérence
🏗️ Clôturer marchés Profiter de la reprise pour clôturer 🧹 Épuration
💡 Conseil pratique : Mieux vaut épurer au maximum pour démarrer sereinement que de reprendre des volumes importants de données obsolètes.

🚨 12. Gestion des Anomalies

🚨 Signal d'Alerte 📝 Description 🔧 Action Corrective
📏 BE > 4000 lignes Volume trop important pour l'outil standard Solliciter équipe technique spécialisée
❌ Erreurs BATMAN Problèmes format/cohérence Corriger avant tentative import
⏰ Retards planning Dérive calendaire Reprioriser/Escalader
💰 Incohérences 408 BE < SF reprise Vérifier correspondances
🚫 API tranches Limitation technique actuelle Import manuel obligatoire

🔧 Procédures d'Escalade

🚨 En cas de blocage majeur : Escalader immédiatement vers l'équipe technique et documenter précisément le contexte d'erreur.

⚠️ Niveau 1 - Équipe projet :

  • Erreurs de format corrigeables
  • Problèmes de cohérence mineurs
  • Retards < 3 jours

🚨 Niveau 2 - Support technique ELAP :

  • BE > 4000 lignes
  • Problèmes API/Webservices
  • Blocages techniques majeurs

🔥 Niveau 3 - Direction projet :

  • Retards > 1 semaine
  • Remise en cause stratégie
  • Problèmes de ressources

📖 13. Lexique et Abréviations

🔤 Abréviations Principales

🏷️ Abréviation 📝 Terme Complet 🔍 Définition
AC Accord-Cadre Contrat public permettant de sélectionner fournisseurs pour commandes ultérieures
AE Autorisations d'Engagement Limite supérieure des dépenses pouvant être engagées juridiquement au cours de l'exercice
BE Bilan d'Entrée (ou Balance d'Entrée) Reprise des soldes comptables de l'ancien système vers ELAP Finance
CP Crédits de Paiement Limite supérieure des dépenses pouvant être payées pendant l'exercice
DP Demande de Paiement Acte administratif pour demander au comptable public de procéder au paiement d'une dépense
DPA Demande de Paiement d'Avance Document pour ordonner le versement anticipé d'une somme à un fournisseur
DRV Demande de Reversement Document pour ordonner le reversement d'une somme indûment perçue
DV Demande de Virement Document permettant le versement d'une subvention ou d'une dotation
EJ Engagement Juridique Acte créant une obligation légale ou contractuelle envers un tiers
GBCP Gestion Budgétaire et Comptable Publique Cadre réglementaire de la comptabilité publique (décret n°2012-1246)
MCAP Mandat de Charge à Payer Mandat émis pour comptabiliser une charge avant réception de facture
OCB Organisme soumis au Contrôle Budgétaire Établissement avec contrôle a priori des dépenses par le comptable public
OHCB Organisme Hors Contrôle Budgétaire Établissement sans contrôle a priori, contrôle a posteriori uniquement
RDD Reprise De Données Processus de migration des données de l'ancien vers le nouveau système
SF Service Fait Constatation de la réalisation d'une prestation ou livraison d'un bien
TIT Titre de Recette Document émis pour constater une créance et ordonner son recouvrement

📚 Glossaire Technique

🏷️ Terme 📝 Définition
CAPAC Charge à Payer Avec Commande - Mécanisme comptable permettant de constater une charge avant réception de la facture, nécessitant une extourne au 1er janvier
DP directe Demande de Paiement non rattachée à un engagement juridique, consommant directement la charge budgétaire
Exercice de reprise Exercice technique (N-1) créé spécifiquement pour accueillir les données de l'ancien système, sans budget ni contrôles
Format pivot Format standardisé ELAP permettant l'import des données, obtenu par transformation du format source client
Format source Format initial des données client, dépendant de leur système d'origine, nécessitant transformation
Lettrage Association comptable entre écritures (ex: SF et paiement) permettant le suivi et l'équilibrage
Pluriannualité Capacité des objets de gestion à être utilisés sur plusieurs exercices (DP 2026 sur SF 2025)
Reste à certifier Montant d'un engagement non encore constaté par un service fait (EJ - SF)
Reste à liquider Montant d'un engagement non encore payé (EJ - DP ou SF - DP)
Tranche conditionnelle Partie optionnelle d'un marché, activable selon décision et budget disponible
